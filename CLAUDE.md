# CLAUDE.md — 잔소리 필터기 프로젝트 가이드

이 파일은 Claude Code가 이 프로젝트에서 작업할 때 자동으로 읽는 지침 파일이다.
프로젝트 루트에 `CLAUDE.md`라는 이름으로 두면 세션 시작 시 자동으로 컨텍스트에 로드된다.

---

## 0. 공통 철학 (혼잡스 자동화 기업)

- 로그인·회원가입이 필요 없는 "초간단 1페이지 웹 도구"를 만든다.
- 사용자의 명확한 불편함(고통)을 해결하되, 체류 시간을 확보할 수 있는 요소를 반드시 포함해 애드센스 광고 수익을 극대화하는 것이 목표다.
- 이 프로젝트: **잔소리 필터기** — 상사 잔소리를 순화하고 업무 요약을 뽑아주는 도구. 완전 클라이언트 사이드(외부 API·백엔드 없음).

---

## 1. 로컬 연결 (Claude Code)

- 이미 설치·로그인 완료된 상태로 사용 중. 재설치 불필요.
- 실행 환경: **Windows**, 셸은 PowerShell(주) + Git Bash(POSIX 스크립트용) 병행.
- 작업 디렉토리: 이 프로젝트 루트(`C:\Users\LG\Desktop\클로드`) 기준으로 실행.

---

## 2. 필수 세팅

### 설정 파일 위치

| 파일 | 용도 |
|------|------|
| `~/.claude/CLAUDE.md` | 전역 지침 (모든 프로젝트 공통) |
| `<프로젝트>/CLAUDE.md` | 프로젝트별 지침 (이 파일) |
| `~/.claude/settings.json` | 전역 설정 (권한, 훅, 환경변수) |
| `<프로젝트>/.claude/settings.local.json` | 개인 설정 (git 제외) |

### 배포 규칙 (중요)

- **배포는 `git push origin main`으로만 한다.** GitHub(`teoteobak-commits/teoteobak`)에 연결된 Vercel이 자동으로 재배포한다.
- Vercel CLI/드래그앤드롭으로 직접 배포하지 않는다 — 기존 자동배포 파이프라인을 그대로 유지한다.
- 배포·호스팅 구성을 임의로 바꾸지 않는다.

## 3. 하네스(Harness) 기본 내용

### Claude가 쓰는 주요 도구

- **Read / Write / Edit** — 파일 읽기·생성·수정
- **Bash / PowerShell** — 셸 명령 실행
- **Glob / Grep** — 파일·코드 검색
- **Agent(서브에이전트)** — 큰 탐색·병렬 작업 위임용 (이 프로젝트는 현재 실제 서브에이전트 팀 기능을 세팅하지 않은 상태 — "기획 팀장/디자인 팀장/개발 팀장" 표기는 보고 형식일 뿐, 기술적으로는 단일 세션에서 순차 처리됨)
- **Artifact / Browser 도구** — 결과물 미리보기, 실제 렌더링 검증

### 컨텍스트 로딩 순서

1. `~/.claude/CLAUDE.md` (전역 지침)
2. 프로젝트 루트의 `CLAUDE.md` (이 파일)
3. 메모리 인덱스 (`~/.claude/projects/.../memory/MEMORY.md`)

## 4. 작업 원칙

우선순위: **정확성 > 검증 > 최소 변경 > 명확성 > 유지보수성**

- 파일·코드가 존재한다고 가정하지 말고 먼저 읽어서 확인한다.
- 요청된 작업에만 변경을 국한하고, 관련 없는 리팩토링은 하지 않는다.
- 가장 단순한 해결책을 선호하고, 불필요한 의존성·추상화를 추가하지 않는다.

### 절대 검증 규칙 (거짓말 방지)

모든 작업은 완료했다고 보고하기 전에 **최소 2회 이상 직접 검수**해야 한다.
- 코드/문법 오류가 없는지 정적으로 재확인 (1차)
- 실제 브라우저에서 열어 동작·화면을 직접 확인 (2차, 스크린샷 등 시각적 증거 포함)
- "다 됐다"고만 하지 말고, 검수 결과와 증거를 그대로 제시한다.
- 확인하지 못한 부분은 확인하지 못했다고 그대로 말한다.

## 5. 부서별 업무 수칙

1. **기획**: 유저의 고통 지수가 높은 문제 정의, 직관적 UI 구조, 체류 시간을 늘리는 요소 기획
2. **디자인**: 다크모드 톤앤매너 유지, 최신 트렌드에 맞는 깔끔한 레이아웃
3. **개발**: 코딩을 모르는 관리자가 그대로 쓸 수 있도록 **단일 `index.html` 파일**로만 작성. 외부 API·백엔드 연동 없이 브라우저에서 완결.
